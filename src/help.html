What is TranscriptDB?

TranscriptDB is a personal, locally-run search engine for your Future Fiction Academy content — weekly summit recordings and Teachable course material. Everything lives on your machine in a SQLite database. It ships as a desktop app for Mac, Windows, and Linux.

What's in it

The database holds two types of content side by side:

  • Summit transcripts — imported from JSON files. These are the weekly FFA Publishing Summit recordings: pre-lesson Q&As, post-lesson Q&As, full lessons, and work sessions.
  • Teachable course content — scraped directly from future-fiction-academy.teachable.com after you log in. The text of each lecture is extracted and stored for searching.

How it works

All content is split into small overlapping chunks and indexed using SQLite's built-in full-text search (FTS5). When you search, the database finds the most relevant chunks instantly — no network required.

The AI Search feature goes further: it retrieves the most relevant chunks and sends them to an AI model via OpenRouter, which synthesizes a direct answer with citations.

Course lectures can also have their Hotmart-hosted videos archived locally. Once archived, an inline player appears in the lecture detail view — and every timestamp in the transcript becomes a clickable link that jumps the video to that moment.

The stats bar

The four numbers at the top of the app tell you what's loaded:

  • Sources — total number of transcript sources + scraped courses
  • Lectures — individual sessions across all sources
  • Transcripts — individual transcript files (a lecture may have multiple: lesson + Q&As)
  • Chunks — the actual indexed text segments used for search and AI context

Browsing Content

When no search is active, the main area shows the Browse view — a grid of all available transcripts and course lectures. The left sidebar is a collapsible file-explorer tree.

The sidebar tree

Teachable courses are grouped at the top level by their tag prefix: CUT, INK, LAN, LED, MID, and Other. Click a group label to expand or collapse it.

Inside each group, courses expand into a hierarchy:

  • Course — click to browse all lectures for that course
  • Section — click to narrow to that section's lectures
  • Class group — click to narrow to a specific class number
  • Individual lecture — click to open the detail view directly

Below the course tree, a Show transcripts ▾ toggle reveals non-course sources such as Summit transcript collections. Click it again to collapse.

Resizing the sidebar

Drag the vertical seam between the sidebar and the main content to widen or narrow the sidebar. The width is saved in localStorage and remembered on your next session.

The lecture grid

Each card in the grid shows:

  • The lecture name
  • The section it belongs to
  • A type badge (Lesson, Pre Q&A, Post Q&A, Work Session, or Course)
  • The date and duration if available

Click any card to open its full transcript in the Detail view.

Notion notes bar

When a Teachable course has a linked Notion page, a View Notes in Notion → bar appears above the lecture grid. Click it to open the notes in a new tab. Click Edit URL to update the link.

Returning to browse

From the Detail or Search view, click the ← Back button or press Escape to return. If you were viewing a search result, Escape goes back to search results; from search results, it returns to browse.

Searching

Full-text search runs entirely on your machine using SQLite FTS5 — it's fast and works offline.

How to search

Type anything into the search bar and results appear automatically after a short pause. Press Enter to search immediately. Press ⌘K (or Ctrl+K) from anywhere to focus the search bar.

Tip: FTS5 uses a Porter stemmer, so searching for "publish" will also match "published", "publishing", "publisher", etc. You don't need to worry about exact word forms.

Search results

Each result card shows the matching chunk with the matched terms highlighted. The result count line beneath the search bar tells you how many matches were found across transcripts and courses.

Click a result card from a transcript source to open the full transcript with your search terms highlighted.

Course lecture results (marked with a Course badge) open the lecture detail view — with the inline video player shown above the transcript if that lecture has an archived video.

Filtering search results

All active filters apply to search results too:

  • Sidebar tree — selecting a course or source in the tree limits results to that content
  • Type chips — limit to one content type
  • Session — limit to a specific lecture/session within the selected source

Clearing search

Click the ✕ button that appears inside the search bar, or clear the text manually. The view returns to Browse.

AI Search

AI Search combines full-text retrieval with an AI language model to give you synthesized answers drawn directly from your content.

Enabling AI Search

Click the ✨ AI button in the search bar to toggle AI mode on. The button turns amber when active and the search placeholder changes to indicate AI mode.

Requires setup: AI Search needs an OpenRouter API key and a selected model. If neither is configured, the settings panel opens automatically when you try to search. See the Settings section for setup instructions.

How it works

  1. You type a question and press Enter (AI mode does not auto-search as you type — it waits for Enter).
  2. The server runs a full-text search to find the most relevant transcript and course chunks.
  3. Those chunks are assembled into context and sent to the selected AI model via OpenRouter.
  4. The model's answer streams back in real time and is displayed in the amber answer card above the results.
Tip: Ask specific questions rather than keyword searches. "What did the instructors say about pricing audiobooks?" works better in AI mode than plain search.

The answer card

The answer card shows:

  • The AI's synthesized answer, streaming in as it is generated
  • The model name used for this response
  • After completion: how many transcript chunks were used as context, and the token count (in / out)

Costs

Each AI search query uses tokens from your OpenRouter account. The cost depends on the model selected. Cheaper models (e.g., Gemini Flash, Llama) cost fractions of a cent per query; premium models (e.g., GPT-4o, Claude) cost more. The token count shown after each response helps you track usage.

Stopping a query

If you start a new AI search before the previous one finishes, the previous request is automatically cancelled.

Transcript Detail

The Detail view shows the full text of a single transcript or course lecture. When a lecture has an archived video, the inline player appears above the transcript.

What you'll see

  • Video player — appears above the transcript when the lecture has at least one archived video file. Not shown for transcript-only sources or un-archived lectures.
  • Header — lecture name, section, type badge, date, and duration
  • Archive Videos button — visible on course lecture headers; opens the archiving progress modal for that course
  • Full text — the complete transcript content
  • Timestamps — time codes like [00:15:30] or [MM:SS] are rendered as amber clickable links when a player is present (click to seek + play), or styled for visual scanning when there's no player
  • Speaker names — patterns like "Name:" at the start of a paragraph are highlighted to make dialogue easier to follow
  • Clickable links — all URLs in the text are rendered as live links that open in a new tab
  • Search highlights — if you arrived from a search result, your search terms are highlighted throughout the text

Setting a Notion URL from a lecture

If a course lecture's text contains a notion.site link, a Set as course Notion URL button appears below the lecture header. Clicking it saves that URL to the course so it appears in the Notion notes bar on the browse grid.

Navigation

Use the ← Back button at the top left, or press Escape to return to your previous view.

Video Player

When a course lecture has been archived locally, an inline video player appears above the transcript in the detail view. You can seek the video by clicking any timestamp in the transcript.

Where the player appears

The player is only shown in the lecture detail view for Teachable course lectures that have at least one archived video file. It does not appear for summit transcript sources, or for lectures that haven't been archived yet. If a lecture hasn't been archived, the detail view shows transcript text only.

To get the player: Use the Archive Videos button on the course detail header (or the Archive Videos button inside any lecture detail header) to download the video for that course. See the Archiving Videos section for details.

Click-to-seek timestamps

Every [HH:MM:SS] or [MM:SS] timestamp in the transcript text is rendered as an amber clickable link when a player is present. Click any timestamp to:

  • Seek the player to that position
  • Start playback automatically

This is the primary way to navigate a long lecture — scan the transcript for the topic you want, then click the timestamp next to it.

Multi-video tabs

Some lectures contain more than one Hotmart-hosted video (for example, FFA Software Support Class Episodes with multiple segments). When a lecture has multiple archived video files, the player shows tabs above the video: Video 1, Video 2, etc. Click a tab to switch the player to that file. Timestamps in the transcript correspond to whichever video is currently active.

Playback speed

A speed selector appears on the player controls. Available speeds: 0.5×, 0.75×, 1×, 1.25×, 1.5×, 1.75×, 2×. Your chosen speed is saved in localStorage and applied automatically the next time you open any lecture with a player.

Resizing the player

The player has a drag handle in its bottom-right corner. Drag it to make the player taller or shorter. The height is saved in localStorage and remembered across lectures and app restarts.

Sources & Filters

The sidebar tree and filter chips give you several ways to narrow what you see in browse and search.

Sidebar tree — courses

The upper portion of the sidebar shows Teachable courses grouped by tag prefix (CUT / INK / LAN / LED / MID / Other). Click a group to expand it, then click into a course, section, class group, or individual lecture. Clicking a node at any level narrows the main view to that scope.

Sidebar tree — transcripts

Below the course groups, a Show transcripts ▾ toggle reveals all non-course transcript sources (Summit sessions, imported collections, etc.). Click a source to browse it. Click again to collapse the transcripts panel.

Type filter chips

The Type chips filter content by recording type. See the Content Types section for what each type means. Only one type can be active at a time; click All to clear the type filter.

Sessions list

When a transcript source is selected, the Sessions list (inside the source view) shows all available session names. Click a session to narrow the view to only that session. When a Teachable course is selected, the tree itself provides section-level filtering.

Content Types

Summit transcripts are organized by the role they played in each weekly session.

Teachable Courses

TranscriptDB can scrape lecture text directly from your enrolled courses on future-fiction-academy.teachable.com and make them searchable alongside your summit transcripts.

Logging in

Click 🔐 Log in to Teachable in the sidebar. A browser window opens (Puppeteer) for you to log in with your Teachable credentials. Once logged in, the auth indicator in the header turns green and the session is saved so you don't need to log in again during the same session.

Note: Your Teachable session cookies are stored locally in the app's userData directory. They are not committed to git.

Adding a course

  1. Click the + button next to the tree header. A panel slides open showing your enrolled Teachable courses, grouped by tag prefix.
  2. Check the courses you want to add.
  3. Optionally check Force Refresh to re-fetch transcript text for lectures that have already been scraped.
  4. Click Scrape Selected. A progress bar shows the scraping progress lecture by lecture.
  5. When complete, the new course appears in the sidebar tree and is immediately searchable.
Tip: Scraping can take a few minutes for large courses. The progress bar shows the current lecture being processed and the overall percentage. Re-scraping is safe — existing video archives are preserved.

How scraping works

For each lecture, the scraper first tries to download any attached text file (.txt), which gives clean content. If no text file is attached, it extracts the visible text from the lecture page DOM. The text is then chunked and indexed for full-text search and AI context.

The scrape uses an idempotent upsert — lectures are matched by their Teachable ID, so re-scraping updates content without destroying archived video data. Lectures removed from the course are soft-deleted rather than wiped.

Notion URL

Each course can store a link to a Notion page (e.g., notes and transcripts for that course). When set, a View Notes in Notion → bar appears above the lecture grid whenever that course is selected.

There are two ways to set a Notion URL:

  • From the notes bar: click Edit URL (or Add Notion URL if none is set yet) and paste the URL.
  • From a lecture: if a lecture's text contains a notion.site link, a Set as course Notion URL button appears in the detail view. Click it to save that URL to the course.

Deleting a course

Open the course in the sidebar tree and look for the delete option in the course detail header. Confirm to remove the course and all its indexed content from the database. Archived video files in the media library are not deleted automatically.

Archiving Videos

Archiving downloads all Hotmart-hosted lecture videos to your local media library so you can watch them offline — and use click-to-seek timestamps in the transcript.

Requirements

Video archiving requires ffmpeg to be installed on your system. If it's missing, an orange banner appears at the top of the app with install instructions:

  • Mac: brew install ffmpeg
  • Windows: choco install ffmpeg or download from ffmpeg.org
  • Linux: sudo apt install ffmpeg (or distro equivalent)

You also need to have the course scraped (transcript text indexed) before archiving.

How to archive

  1. Open a course by clicking it in the sidebar tree.
  2. Click the Archive Videos button in the course detail header.
  3. A progress modal opens, showing each lecture as it's processed. The scraper visits each lecture page, extracts the Hotmart video URL(s), and downloads them via ffmpeg.
  4. When done, close the modal. Lectures with archived video now show a player in their detail view.
Tip: You can cancel the archive run mid-way using the Cancel button in the progress modal. Already-downloaded lectures are kept. Re-running archive is safe — fully-archived lectures are skipped automatically.

Multi-video lectures

Lectures with more than one Hotmart embed (e.g., Bookcamp episodes with multiple segments) have all their videos downloaded. Files are named video.mp4, video_2.mp4, video_3.mp4, etc. Each file corresponds to a tab in the video player.

Progress modal status labels

StatusMeaning
Downloadingffmpeg is actively writing the file for this lecture.
Skipped (already archived)All expected video files for this lecture already exist on disk — no download needed.
Skipped (wrong provider)This lecture's video is hosted somewhere other than Hotmart (e.g., YouTube). Not supported yet.
No videoThe lecture page had no embedded video to extract.
ErrorSomething went wrong for this lecture. The rest of the run continues.

Where files are stored

Video files go into your Media Library, in a subfolder per course and per lecture. The media library path is set in Settings → Media Library Path. The default is shown during first-run setup. You can change the path at any time — existing files stay where they are, and you'd need to move them manually if you change the path.

Settings

Click the gear icon (⚙) in the top-right header to open Settings. This is where you configure the media library path, AI search, and other preferences.

First-run splash

On first launch, a splash modal appears before the app is usable. It shows the default Media Library Path (where archived videos will be stored) and asks you to confirm it. You can change the path here before confirming. The media library can grow very large if you archive many courses — pick a drive with plenty of space.

You can update this path at any time later in Settings.

Media Library Path

The media library is a folder on your machine (or an external drive) where all archived course videos are stored. Click Choose Folder in Settings to open a native folder picker and select a new location.

Note: Changing the media library path does not move existing files. If you switch to a new path, the app will look for videos in the new location and treat previously-archived lectures as un-archived until you move the files there manually.

OpenRouter API Key

AI Search routes all requests through OpenRouter, which gives you access to hundreds of models from one API key.

  1. Go to openrouter.ai/keys and create a free account.
  2. Generate an API key (starts with sk-or-v1-).
  3. Paste the key into the API key field in Settings and click Save.

The key is stored locally in the app's userData directory and never transmitted anywhere except to OpenRouter when you make AI searches.

Selecting an AI model

After saving your API key, click ↻ Refresh in the Model section to fetch the current list of available models. Then click the model dropdown and choose one. You can filter the list by typing in the search box.

The pricing shown next to each model is per million tokens (input / output). For casual use, models like google/gemini-flash-1.5 or meta-llama/llama-3.1-8b-instruct:free are good starting points.

Tip: Your selected model and playback speed are saved automatically and persist across sessions.
